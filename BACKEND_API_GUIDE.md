# Nexi5 HRM Portal: Backend API & Database Reference

This document provides a highly detailed technical reference for the Nexi5 backend, including API endpoints, JSON request/response schemas, and Firestore database collections.

---

## 🏗️ Architecture Overview
- **Engine**: Node.js/Express.
- **Database**: Google Firestore (NoSQL).
- **Authentication**: Firebase Admin SDK (JWT via Bearer Token).
- **Real-time**: Socket.IO for event broadcasts.

---

## 🗄️ Database Collections & Schemas

### 1. `Staff` (Management Personnel)
Stores profiles for Admin, HR, and Managers.
- **Key**: Firebase UID.
- **JSON Structure**:
```json
{
  "staffId": "STAFF-1711884000000",
  "uid": "FIREBASE_UID_STRING",
  "name": "Full Name",
  "email": "user@gmail.com",
  "role": "Admin | HR Head | Manager | BDE",
  "status": "active",
  "createdAt": "ISO_TIMESTAMP"
}
```

### 2. `employees` (Registered Personnel)
Stores standard employee records with nested data.
- **Key**: Firebase UID.
- **JSON Structure**:
```json
{
  "status": "active",
  "leaveBalance": { "casual": 12, "sick": 10, "annual": 15 },
  "employeeData": {
    "employeeId": "EMP-1711884000000",
    "name": "Employee Name",
    "email": "emp@gmail.com",
    "department": "IT",
    "designation": "Software Engineer",
    "joiningDate": "YYYY-MM-DD",
    "basicSalary": 50000,
    "allowances": 5000
  }
}
```

### 3. `attendance` (Daily Logs)
- **Key**: Auto-generated by Firestore.
- **JSON Structure**:
```json
{
  "employeeId": "FIREBASE_UID_STRING",
  "date": "YYYY-MM-DD",
  "checkin": "ISO_TIMESTAMP",
  "checkout": "ISO_TIMESTAMP",
  "totalHours": 8.5
}
```

## 📡 API Reference & JSON Schemas

### 🔐 Authentication

#### **Register User**
`POST /api/register`
- **Request Body**:
```json
{
  "name": "John Doe",
  "email": "john@gmail.com",
  "password": "securepassword",
  "role": "HR Head"
}
```
- **Response (201)**:
```json
{
  "success": true,
  "message": "HR Head account created successfully.",
  "uid": "GENERATED_UID"
}
```

#### **Login User**
`POST /api/login`
- **Request Body**:
```json
{ "email": "john@gmail.com" }
```
- **Response (200)**:
```json
{
  "success": true,
  "user": { "uid": "UID", "email": "john@gmail.com", "role": "HR Head", "name": "John Doe" },
  "token": "dev-token-UID",
  "dashboardPath": "/dashboard"
}
```

---

### 👥 Personnel Management

#### **Create Employee** (HR only)
`POST /api/employees`
- **Request Body**: Same as `employeeData` schema above.
- **Logic**: Automatically creates Firebase Auth user and sends credentials email.

#### **Unified Users List**
`GET /api/users`
- **Response (200)**: Array of mixed Staff and Employee records.

---

### ⏱️ Attendance & Analytics

#### **Check-In**
`POST /api/checkin`
- **Headers**: `Authorization: Bearer <TOKEN>`
- **Logic**: Validates if already checked in today. Returns 400 if duplicate.

#### **Monthly Analytics**
`GET /api/attendance/advanced/monthly?month=2026-03`
- **Response (200)**:
```json
{
  "success": true,
  "data": [
    {
      "name": "Lokesh",
      "attendancePercentage": 95,
      "avgWorkingHours": 8.2,
      "productivityScore": 88,
      "insights": ["High Performer"]
    }
  ]
}
```

---

### 📂 Company Projects

#### **Create Project**
`POST /api/company-projects`
- **Request Body**:
```json
{
  "projectName": "Nexi5 Setup",
  "clientName": "Internal",
  "description": "System initialization",
  "startDate": "2026-03-31",
  "endDate": "2026-12-31",
  "status": "active"
}
```
- **Socket Event**: Broadway `project_created` to all clients.

## ⛓️ Security & RBAC
The backend uses **`verifyToken`** middleware to:
1. Extract the UID from the Bearer token.
2. Resolve the Role from Firestore (`Staff` or `employees` collection).
3. Attach `{ uid, role, email }` to the `req.user` object.

| Role | Access Level |
|---|---|
| **Admin / HR Head** | Full CRUD on Employees, Projects, and Payroll. |
| **Manager** | View all analytics, manage assigned projects/tasks. |
| **Employee** | View personal attendance, tasks, and slips. |
